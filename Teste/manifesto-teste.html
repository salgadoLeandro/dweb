Resumo

Exercício nº 1: Compositores Musicais

No primeiro exercício foi pedido que se desenvolvesse uma API de dados para um dataset em XML.

Primeiro, o XML foi convertido manualmente, com recurso a um editor de texto (neste caso Visual Studio Code) e Regex, para JSON num formato que pudesse ser importado para MongoDB.

De seguida, o ficheiro resultado foi importado para MongoDB, criando assim a base de dados da aplicação.

Finalmente, foi elaborada um aplicação web em NodeJS que expõe um API de dados, permitindo o acesso à base de dados previamente criada.

De modo a executar a aplicação, é necessário primeiro tornar a base de dados operacional. Para tal, estando no root do projeto (neste caso, directoria Teste), executa-se o daemon do MongoDB com mongod --dbpath data.

Após a base de dados estar operacional, falta apenas instalar os módulos necessários para o funcionamento da mesma. Mudando para a directoria da aplicação (neste caso, Teste/compositores), executa-se o comando npm install, para instalar os módulos, seguido do comando npm start para executar a aplicação.

Exercício nº 2: CLAV: Classificação e Avaliação da Informação Pública

No segundo exercício foi pedido que se repondesse a 3 questões e que se desenvolvesse uma interface web para uma API de dados já existente.

A aplicação foi elaborada em NodeJS com recurso a PUG para facilmente fazer a interface da aplicação.

Uma vez que a base de dados é externa à aplicação, o acesso à mesma foi feita através do package axios.

Assim, após mudar para a directoria da aplicação (neste caso, Teste/clav), executa-se o comando npm install, para instalar os módulos, seguido do comando npm start para executar a aplicação.

As questões e respectivas respostas encontram-se a seguir:

Q: Quantos processos (nível 3) e quais são (obtem uma lista em JSON; podes concatenar sublistas invocando várias queries), existem na descendência da classe 200?

R: 22 processos. A lista está no ficheiro clav_1.json, que está nos resultados.

Q: Quantos subprocessos existem? (classes de nível 4)

R: 116 subprocessos.

Q: Quantos processos (classes de nível 3) se encontram subdivididos em níveis 4?

R: 58 processos nível 3 encontram-se subdivididos em níveis 4.
